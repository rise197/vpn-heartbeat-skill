# VPN Heartbeat Skill

## 触发条件

当 AI 助手首次加载此 Skill 时，**必须立即**对单点登录系统发起认证请求。

## 执行流程

```
AI 加载 Skill
  │
  ▼
┌──────────────────────────────────────────┐
│ 1. 构建并运行 Go 二进制                    │
│    cd <skill-dir> && go build -o vpn-skill ./cmd/skill/ && ./vpn-skill
│    或使用预编译二进制: ./vpn-skill
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│ 2. Go 引擎自动执行:                        │
│    a. 读取 config.yaml 站点配置            │
│    b. 按优先级尝试 SSO 节点认证             │
│       · https://query-test.ccdc.com.cn    │
│       · https://106.38.112.120            │
│    c. 认证成功后并发登录所有业务站点         │
│    d. 启动 60s 间隔心跳维持会话             │
│    e. 输出 JSON 状态到 stdout              │
└──────────────┬───────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────┐
│ 3. 解析输出确认结果                        │
│    ✓ 所有站点认证成功 → 报告状态            │
│    ✗ 部分失败 → 报告失败站点和原因          │
└──────────────────────────────────────────┘
```

## 首次加载时必须执行的操作

1. 在 Skill 目录下执行 `go build -o vpn-skill ./cmd/skill/ && ./vpn-skill -config config.yaml`
2. 等待输出结果
3. 如果 SSO 认证成功，报告: "VPN 心跳 Skill 已就绪，N 个站点认证成功"
4. 如果 SSO 认证失败，报告: "SSO 认证失败: <原因>"
5. 将 `vpn-skill` 进程保持在后台运行以维持心跳

## 配置

所有站点配置在 `config.yaml` 中，包括:
- SSO 端点地址和凭据
- 心跳间隔（默认 60s）
- 各业务站点 URL、账号、密码

## 心跳输出格式

Go 引擎每隔 60s 输出一次状态:

```json
{
  "heartbeats": 42,
  "active_sessions": 5,
  "site_health": {
    "银登网": "ok",
    "文件服务上传管理系统": "ok",
    "商业银行理财业务信息披露平台": "ok",
    "中国理财网": "ok",
    "单点登录系统": "ok"
  }
}
```
